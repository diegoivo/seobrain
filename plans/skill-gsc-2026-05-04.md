## Objetivo

Adicionar conexão com Google Search Console ao SEO Brain, complementando o Pilar Dados (que hoje só tem DataForSEO, pago e baseado em estimativas) com **dados reais e gratuitos** do próprio site do cliente: queries que trazem tráfego, posição média, CTR real, indexação, sitemaps, erros.

GSC é a fonte primária de SEO autoritativa. Sem ele, o kit opera cego sobre o desempenho real.

## Decisões já fechadas (discussão com Diego)

1. **Caminho A — BYO credentials.** Cliente cria projeto Google Cloud + OAuth client próprio. Sem app verification, sem cap de 100 usuários, credenciais isoladas por projeto.
2. **Setup manual guiado, sem automação de browser.** Agente abre URLs no Chrome real do usuário via `open -a "Google Chrome" <url>` e instrui passo a passo. Sem gstack, sem MCP browser, sem CDP attach. Robusto, funciona em qualquer harness, sessão Google do usuário já está ativa.
3. **3 skills no MVP:** `/gsc-setup`, `/gsc-performance`, `/gsc-coverage`.
4. **Padrão de output triplo idêntico ao DataForSEO:** `.md` + `.csv` + `.json` em `brain/seo/data/gsc/`.
5. **Lib oficial `googleapis` (npm).** Não implementar OAuth na mão.

### Variáveis de ambiente (`.env.local`)

```bash
# Google Search Console (skill /gsc-*)
# Ver /gsc-setup para gerar estas credenciais.
GSC_CLIENT_ID=
GSC_CLIENT_SECRET=
GSC_REFRESH_TOKEN=
GSC_PROPERTY=https://exemplo.com.br/   # ou sc-domain:exemplo.com.br
```

`GSC_PROPERTY` aceita os 2 formatos do GSC: URL-prefix (`https://...`) ou Domain property (`sc-domain:...`).

### Fluxo OAuth do `/gsc-setup`

OAuth Desktop App + loopback redirect (padrão Google para CLI).

1. Verifica `.env.local` — se já tem `GSC_REFRESH_TOKEN`, pergunta se quer reconfigurar
2. Inicia servidor HTTP local em porta aleatória (via `get-port`)
3. **Passo 1:** Abre `https://console.cloud.google.com/projectcreate` no Chrome → instrui criar projeto `seobrain-<slug>`
4. **Passo 2:** Abre `https://console.cloud.google.com/apis/library/searchconsole.googleapis.com?project=<id>` → instrui Enable
5. **Passo 3:** Abre `https://console.cloud.google.com/apis/credentials/consent?project=<id>` → instrui External + nome do app + email + Save
6. **Passo 4:** Abre `https://console.cloud.google.com/apis/credentials?project=<id>` → instrui Create Credentials → OAuth Client ID → **Desktop app** → Create
7. **Passo 5:** Instrui copiar `client_id` + `client_secret` da modal e colar no terminal
8. Constrói consent URL com scope `https://www.googleapis.com/auth/webmasters.readonly` + `redirect_uri=http://localhost:<porta>` + `access_type=offline` + `prompt=consent`
9. Abre URL no Chrome → usuário consente
10. Servidor local captura `code` no callback
11. Troca `code` por `refresh_token` (POST oauth2/v4/token)
12. Lista properties acessíveis via `sites.list` → usuário escolhe qual property registrar
13. Salva tudo em `.env.local` + atualiza `brain/config.md` campo `gsc_property`
14. Mata servidor local
15. Sumário: "✅ GSC configurado. Próximos passos: /gsc-performance, /gsc-coverage."

### Endpoints API usados

| Skill | Endpoint | Cota |
|---|---|---|
| `/gsc-setup` | `sites.list` | irrelevante |
| `/gsc-performance` | `searchanalytics.query` | 1.200 req/min/projeto |
| `/gsc-coverage` | `sitemaps.list`, `sitemaps.get` | 1.200 req/min |

Token refresh: `POST oauth2.googleapis.com/token` com `grant_type=refresh_token`. Cache do access_token em memória durante a invocação (TTL 1h).

### `/gsc-performance` — pipeline

Inputs:
- `--days=90` (default)
- `--limit=100` (default, max 25.000)
- `--dimension=query|page|both` (default `both`, gera 2 outputs)
- `--opportunities` (filtra queries em pos 5-15 com CTR < benchmark)

Output `performance-<date>-queries.md`:

```markdown
# GSC Performance — <date>

Property: https://exemplo.com.br/
Período: últimos 90 dias (2026-02-04 → 2026-05-04)
Total queries: 1.247

## Top queries por cliques

| # | Query | Clicks | Impressões | CTR | Posição |
|---|---|---|---|---|---|
| 1 | seo agêntico | 412 | 8.230 | 5.0% | 3.2 |
| ... |

## Oportunidades (pos 5-15, CTR baixo)

| Query | Pos atual | Impressões | CTR | Estimativa de upside |
|---|---|---|---|---|
| llm wiki | 8.4 | 2.100 | 1.2% | +180 clicks/mês se subir pra top 3 |
```

CSV: tabela bruta. JSON: payload completo + metadata.

### `/gsc-coverage` — pipeline

Lista sitemaps via `sitemaps.list`, agrega por status. Output:

```markdown
# GSC Coverage — <date>

Property: https://exemplo.com.br/

## Sitemaps submetidos

| Path | Última leitura | URLs descobertas | Erros |
|---|---|---|---|
| /sitemap.xml | 2026-05-03 | 142 | 0 |
| /sitemap-blog.xml | 2026-05-03 | 89 | 2 |

## Erros agrupados

| Tipo | Quantidade | Exemplos |
|---|---|---|
| 404 in sitemap | 2 | /blog/post-removido, ... |
```

URL Inspection API (status de indexação per-URL) **não entra no MVP** — fica pra `/gsc-inspect-url` futuro (cota 2000/dia separada).

### Token refresh + error handling

`scripts/gsc-client.mjs::getAccessToken()`:
- Cache em memória do access_token + expires_at
- Se expirado ou ausente: POST `oauth2.googleapis.com/token` com refresh_token
- Erros tratados:

| Erro | Ação |
|---|---|
| `.env.local` sem GSC_* | Aborta com instrução: "Rode /gsc-setup" |
| 401 invalid_grant | "Refresh token expirado/revogado. Rode /gsc-setup novamente." |
| 403 insufficientPermissions | "Conta sem acesso à property `<X>`. Verifique GSC → Settings → Users." |
| 429 rateLimitExceeded | Backoff exponencial 4 retries (250→2000ms) |
| 400 bad property format | "GSC_PROPERTY deve ser URL completa (https://exemplo.com/) ou sc-domain:exemplo.com" |

## Riscos conhecidos

1. **Setup OAuth é a fricção real.** ~5min na primeira vez, mesmo guiado. Mitigação: instruções enxutas + URLs pre-preenchidas com `?project=<id>` quando possível.
2. **GSC tem 2-3 dias de delay.** Não é tempo real. Documentar nas skills.
3. **Anonymized queries.** GSC esconde queries com poucos cliques (privacidade). Aparecem como `(other)` no `/gsc-performance`. Documentar.
4. **`googleapis` lib é grande.** ~10MB. Aceitável para framework dev (não é runtime do site do cliente).

## Notas de implementação

- `googleapis` lib expõe `google.searchconsole('v1')` — usar essa interface, não chamar REST direto
- Token refresh: a própria lib gerencia se passarmos OAuth2Client com refresh_token configurado
- Property no formato `sc-domain:` precisa de URL-encoding ao passar como path param
- `searchanalytics.query` retorna até 25.000 rows; paginação via `startRow` se precisar mais
